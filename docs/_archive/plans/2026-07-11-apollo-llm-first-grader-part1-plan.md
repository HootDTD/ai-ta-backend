## Spec corrections (reality-check vs origin/staging = 691a888)

1. **`compute_coverage` returns FOUR keys, not three.** Spec §3.1 lists `per_step` / `procedure_scores` / `confidences`; the real return (coverage.py:577-586) also includes **`negotiation_counts`** (`{dual, disputed, paraphrased, skipped}` int map), consumed by `diagnostic.py:139`. The frozen contract MUST include it, and `compute_transcript_coverage` must emit it (all-zero counts — the transcript grader has no negotiated student graph). Omitting it silently blanks the negotiation line in the served diagnostic.

2. **Flag-reader precedent is split.** Two patterns exist: local `os.environ` helpers in done.py:199-221 (`.lower() in ("1","true","yes")`) AND module-level readers in `apollo/overseer/misconception_detector/config.py` (`_TRUTHY = frozenset({"1","true","yes","on"})`, read at call time). Since `topic_score_served_enabled()` — the flag the spec says to sit beside — lives in config.py, `transcript_grader_enabled()` goes in **config.py** next to it, imported into done.py.

3. **done.py:574's `coverage` feeds more than the grade.** The variable also flows into `compute_rubric` (:598), `generate_diagnostic` (:806), `write_artifacts(coverage=...)` (:1069), `student_response["coverage"]`, and `attempt.diagnostic_report["coverage"]`. All consume only the four contract keys, so a same-shape swap is safe — but the integration task swaps the **variable source**, not just the grade input.

4. **Shadow lane does NOT consume the outer `coverage`.** Resolved from code: `run_graph_simulation` (done_grading.py:344-515) computes its own coverage internally (`load_problem_candidates_with_soundness` → `grade_attempt` → `build_audited_grade`) and never reads the dict built at :574. So flag-ON can drop `compute_coverage` from the served path without touching the shadow lane — spec §3.5 confirmed correct.

## Task 4 — `transcript_coverage.py`: the structured-output LLM call + mapping into the frozen contract

**Goal:** the single adjudication call and the mapping from `NodeVerdict`s into the frozen `CoverageVerdict` dict (Task 1 contract).

**File to modify:** `apollo/overseer/transcript_coverage.py` (final part).

Add:
- `_call_adjudication(system_prompt, user_message, *, model) -> str` — one `OpenAI().chat.completions.create(model=model, response_format={"type":"json_schema","json_schema": build_transcript_grader_schema()}, messages=[...], temperature=0.0)` returning `resp.choices[0].message.content or "{}"`. This is byte-for-byte the `apollo/parser/parser_llm.py::_call_extraction` pattern (parser_llm.py:299-318), reusing the same `response_format` / `temperature=0.0` shape and the schema-builder helper convention. `model = os.getenv("MAIN_MODEL", "gpt-4o")` (matches parser_llm.py:364).
- `async def compute_transcript_coverage(transcript, reference_graph, problem) -> dict` (signature per spec §3.1; `transcript` is a `Sequence[tuple[str,str]]` of `(role, content)` or a small `TranscriptMessage` dataclass — do NOT depend on the SQLAlchemy `Message` model, keep the module DB-free and unit-testable). Flow:
  1. Build rubric items + prompts; extract `student_messages = [c for (r,c) in transcript if r == "student"]`.
  2. Call `_call_adjudication` via `asyncio.to_thread` (the OpenAI client is sync, mirroring `compute_coverage`'s `asyncio.to_thread` usage).
  3. Parse JSON → per-node raw verdicts; `_quantize_credit`; build `NodeVerdict`s.
  4. **Span validation rail (§3.2):** for each credited (`credit > 0`) verdict, if `not validate_span(evidence_span, student_messages)` → set `credit = 0.0`, `covered = False`, `unverified = True`. This is the anti-over-credit / anti-injection rail.
  5. `_to_coverage_verdict(verdicts, reference_graph) -> CoverageVerdict`: map into the frozen contract:
     - `per_step[node_id]` = `"covered"` iff `covered and credit >= 1.0`, else `"missing"` (mirrors coverage.py's binary semantics; partial lands in `procedure_scores`).
     - `procedure_scores[node_id]` = `credit` for **every** graded node (so `_credit_for_node` reads partials for all four types, matching how topic_score derives partial credit — the current coverage only populates procedure_scores for procedure_step, but topic_score.py:135 reads it for any node_id present, so this is a strict superset that stays contract-valid and gives partial credit to equations/conditions too, which is the spec's intent "credit semantically equivalent statements regardless of node type").
     - `confidences[node_id]` = `confidence`.
     - `negotiation_counts` = `{"dual":0,"disputed":0,"paraphrased":0,"skipped":0}` (transcript grader has no negotiated student graph).
  6. `validate_coverage_verdict(result)` before returning (fail loud on a contract break — NO silent shape drift).
- Terminal-failure posture: raise `CoverageGradingError` (import from `apollo.errors`, same as coverage.py) on unparseable/empty LLM output, so the served path fails loud rather than fabricating a grade. (done.py integration decides fallback, Task 6.)

**Tests to write FIRST** (`apollo/overseer/tests/test_transcript_coverage.py`, `@patch("apollo.overseer.transcript_coverage.OpenAI")`):
- `test_full_credit_maps_to_covered_per_step` — verdict `{covered:true, credit:1.0, span:<verbatim student quote>}` → `per_step[id]=="covered"`, `procedure_scores[id]==1.0`.
- `test_partial_credit_maps_to_missing_per_step_with_procedure_score` — `credit:0.7` → `per_step=="missing"`, `procedure_scores[id]==0.7`.
- `test_span_not_in_student_text_downgrades_to_zero_and_unverified` — credited verdict with a span present only in an apollo turn (or absent) → credit 0, `per_step "missing"`, `procedure_scores 0.0`.
- `test_prompt_injection_span_fabrication_gets_zero` — a span quoting an "instruction" that never appears in student text → downgraded.
- `test_output_satisfies_frozen_contract` — `validate_coverage_verdict(result)` passes; keys exactly the four contract keys; `negotiation_counts` all-zero.
- `test_empty_llm_output_raises_coverage_grading_error`.
- `test_credit_quantized` — a raw `0.55` credit from the model is snapped.
- `test_single_llm_call` — assert `create` called exactly once (one adjudication call per Done, spec §3.2).

**Verification:** `pytest apollo/overseer/tests/test_transcript_coverage.py -v --tb=short`

## Task 8 — Per-topic dock arithmetic (spec §3.3.1) — OWN task

**Goal:** change `compute_topic_score` from global `clamp01(coverage_component − dock)` (topic_score.py:325) to per-topic docks: `score = Σ_i weight_i × max(0, credit_i − dock_i)`, each finding's dock landing on the topic it attaches to (capped at that topic's weight-normalized credit contribution), with a global cap = `SEVERITY_CLAMP` retained ONLY for `_general`-bucket (unattached) findings.

**File to modify:** `apollo/overseer/topic_score.py` — rework the dock section (topic_score.py:288-330). New arithmetic:
- Compute per-topic raw dock from findings whose `_topic_key_for_finding` maps to that topic (using `_finding_penalty_share`).
- Per-topic contribution: `weights[nid] * max(0.0, credits[nid] - per_topic_dock[nid])` (the dock is subtracted **inside** the per-topic term, capped so a topic never goes below 0 — the `max(0,...)` does this).
- `_general` (unattached) findings: sum their shares, clamp to `SEVERITY_CLAMP`, subtract from the total once (preserves today's global-cap semantics for findings that localize nowhere).
- `score = round(clamp01(Σ per-topic contributions − general_dock) * 100)`.
- Keep `TopicMisconception.dock_points` display values consistent with the amount actually subtracted (scale the same way the current clamp-scaling does, but now per-topic). Preserve `_finding_resolved` returning `False` (Part 2 wires resolution — do NOT touch it here).
- `coverage_component` and `misconception_dock` fields on `TopicScoreResult` retained for artifact/telemetry parity: set `coverage_component = Σ weights*credits` (undocked) and `misconception_dock = total actually subtracted`.

**Tests to write FIRST** (`apollo/overseer/tests/test_topic_score_per_topic_dock.py`; also extend existing `test_topic_score.py` where a legacy assertion changes):
- `test_attempt_47_shaped_nonzero` — reference graph shaped like attempt 47: ~30.4% coverage spread across topics, one finding with penalty share ~0.3 attaching to ONE topic. Assert final score is **non-zero** (the earned credit on other topics survives; only the docked topic loses) and in the low band (spec: ~26% F, not 0). Pin the score with `pytest.approx` at the topic-status level per Risk row 3.
- `test_perfect_coverage_with_dock_on_one_topic` — all topics credit 1.0, one topic docked by its full weight → that topic contributes 0, others contribute fully; score < 100 by exactly that topic's weight.
- `test_unattached_finding_uses_global_severity_clamp` — a finding mapping to `_general` docks globally, capped at `SEVERITY_CLAMP`; two such findings summing above the cap are clamped.
- `test_per_topic_dock_clamps_at_zero_not_negative` — a dock exceeding a topic's credit floors that topic's contribution at 0 (no negative, no bleed into other topics).
- `test_dock_points_display_sums_to_amount_subtracted` — the displayed `dock_points` across topics sum to `misconception_dock`.
- Update any existing `test_topic_score.py` case that asserted the **old global** arithmetic (e.g. cases where a dock on one topic previously reduced the whole score) — re-derive expected values under the per-topic rule and note the change in the test docstring.
