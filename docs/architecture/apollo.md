# Apollo — Learning-by-Teaching Subsystem

Apollo inverts tutoring: the student explains a concept, Apollo plays a learner that knows
nothing, and at "I'm done teaching" the explanation is graded by diffing the student's
knowledge graph (KG) against an authored reference graph. State is split across two stores:
Postgres (Supabase) owns sessions/messages/attempts/progress; Neo4j owns the per-attempt KG
subgraph. The current architecture is "diff-at-Done v1" (spec: workspace
`docs/superpowers/specs/2026-06-09-apollo-diff-at-done-design.md`, outside this repo):
the per-turn loop is just **nodify + dumb reply**; the output filter and SymPy solver were
unwired (code still present), and grading is the Done-time LLM semantic diff.

| `apollo/resolution/` | `candidates.py`, `result.py`, `tiers.py`, `structural.py`, `competition.py`, `assignment.py`, `adjudication.py`, `resolver.py` | The **§5 shared reference-anchored resolver** (WU-3C2), STANDALONE so WU-4A's `graph_compare` imports it (`from apollo.resolution import resolve_attempt`) rather than owning it. `resolve_attempt(student_graph, candidates, *, llm_adjudicator=None, fuzzy_threshold=0.9, symbolic_mappings=None) -> ResolutionResult` maps each student evidence node onto a CLOSED candidate set (this problem's reference nodes + course `misc.*`, built by `build_candidate_set` + the `candidates_from_reference_solution`/`candidates_from_misconceptions` adapters) via content-first tiers (exact → SymPy-symbolic REUSING `sympy_exec.parse_zero_form`, sign-exact → normalized alias → RapidFuzz `token_set_ratio` ≥ 0.9, the only RapidFuzz site). **Symbolic mappings are PER-PROBLEM declared data, NOT a global default:** `symbolic_mappings` defaults to `None` → `{}` (no substitution); the §6.9 `d=2r` circular-area equivalence resolves only when the caller passes `symbolic_mappings={'d':'2*r'}` (WU-4A supplies the per-problem table). The structural / anti-over-normalization signals the resolver actually wires are the **type-compat HARD constraint** (`condition↔condition` only) + **misconception competition** ranked on **RAW lexical proximity** (1.0 for an exact-alias hit, the `token_set_ratio` for a fuzzy hit — the §3 method cap is the *reported* confidence, re-derived downstream, never the competition signal): `tiers.match_alias_all`/`match_fuzzy_all` surface EVERY type-compatible above-threshold candidate (each carrying its raw score + winning alias) so a competing misconception is never discarded before competition (a polar near-miss out-competes the lexically-close reference node, §6.11), and a **per-winning-alias polarity/direction screen** rejects only a direction-inverted match on the alias that produced the fuzzy hit (an unrelated inverted alias on the same candidate no longer false-rejects a valid match; misconceptions are exempt — they are supposed to be polar). Each candidate enters the lexical competition **once**: `_content_match` dedupes by candidate and keeps the higher tier EXPLICITLY (alias 0.92 > fuzzy 0.80), so a candidate that exact-alias-matches (raw 1.0) *and* fuzzy-matches itself (token_set_ratio of identical strings == 1.0) reports `alias` deterministically, never order-/tie-dependent on the competition list. **Neighborhood corroboration (§5 steps 2-3) is DEFERRED** — v1 wires type-compat + competition only; the `propagate_and_veto` seam is unbuilt (a WU-4A-era refinement), so `structural.py` carries ONLY `type_compatible` + the `ScoredMatch` dataclass (no neighborhood-boost helper). Then **bounded greedy global assignment** (many students→one reference allowed; one student never splits; cap 150 student nodes → whole-attempt abstention, no hang) + **ONE LLM adjudication** for the remainder via `agent/_llm.main_chat` ("return empty when unsure"; ≤1 call/attempt; hallucinated key → `ResolutionInvalidOutputError`). Per-node output `{resolution, resolved_key, resolution_method, resolution_confidence}` with confidence caps by method (exact 1.0 / symbolic 0.98 / alias 0.92 / fuzzy 0.80 / LLM 0.75 / unresolved 0.0, the §3 damper input). PURE + deterministic given a deterministic adjudicator (re-runs yield identical results); a non-match is `unresolved` DATA (no edge, no exception). The resolver does NOT grade, simulate, persist, or orchestrate Done — matching only (persistence is `resolution_store.py`; grading is WU-4A). |

Key service entry points:
- `parse_utterance(utterance, *, concept: ConceptDefinition, attempt_id: int, graph_context: GraphContext | None = None, model=None) -> tuple[list[Node], list[Edge]]` — one strict-json_schema GPT-4o call → typed nodes + all four typed edges (provenance-tagged); `graph_context` (optional) supplies prior-attempt nodes for cross-turn edge linking — `chat.py` builds and threads it on every turn (WU-2B). Raises `ParserCouldNotExtractError` only when a *non-trivial* utterance yields zero entries.
- `build_graph_context(graph: KGGraph) -> GraphContext` (pure, `apollo/parser/graph_context.py`) — projects a read subgraph into the parser's prior-attempt context, deriving a short per-type label and skipping any id that would collide with the `^n\d+$` ordinal namespace (`is_safe_context_id`).
- `KGStore.write_nodes(*, attempt_id, nodes, source, user_id=None, search_space_id=None) -> int` (count CREATED; cross-turn-reused ids excluded; WU-3C1 scoping kwargs are keyword-only + default `None` = back-compat, injected as Layer-2 node props plus an always-present `created_at`) / `write_edges(*, attempt_id, edges, source) -> WriteEdgesResult` (`written`/`dropped`/`invalid`/`reasons`; `int()`-coercible to `written` for back-compat; validates before CREATE, logs every rejection) / `read_graph(*, attempt_id) -> KGGraph` / `delete_subgraph(*, attempt_id)` / `stamp_graded_at(*, attempt_id) -> int` (WU-3C1; sets `graded_at` on the frozen subgraph, idempotent, raises `RetentionError` on Neo4j failure) / `summarize_for_apollo(*, attempt_id) -> str` / `freeze(session_id)` / `unfreeze(session_id)` / `mark_node_disputed` / `paraphrase_node` / `skip_node` / `get_node_trace`.
- `apollo.knowledge_graph.canon_projection.project_canon(db, neo, *, search_space_id=None, concept_id=None) -> CanonProjectionResult` (WU-3C1; idempotent `:Canon` MERGE from Postgres entities; raises `CanonProjectionError`; refuses an unscoped seed via `ValueError`) — pure helpers `entity_to_canon_spec`/`canon_spec_to_row`, async `load_entity_specs`/`merge_specs`. CLI: `scripts/seed_canon_projection.py`.
- `apollo.resolution.resolve_attempt(student_graph, candidates, *, llm_adjudicator=None, fuzzy_threshold=0.9, symbolic_mappings=None) -> ResolutionResult` (WU-3C2; the §5 resolver — pure + deterministic; per-node `ResolvedNode{node_id, resolution, resolved_key, resolved_canon_key, method, confidence}`; `ResolutionResult.resolved_edges()` yields the resolved-with-`:Canon`-key subset; `result.llm_calls ≤ 1`; raises `ResolutionInvalidOutputError` on a hallucinated LLM key, never raises for a non-match). `symbolic_mappings` is PER-PROBLEM declared data — `None` → `{}` (no global default; the §6.9 `d=2r` case resolves only when the caller passes it). Candidate builders `build_candidate_set`/`candidates_from_reference_solution`/`candidates_from_misconceptions`; `METHOD_CONFIDENCE_CAP`. The CI-safe default makes NO live LLM call — the real one-call path is `apollo.resolution.adjudication.main_chat_adjudicator`.
- `apollo.knowledge_graph.resolution_store.write_resolution(neo, attempt_id, result, *, resolved_at) -> ResolutionWriteResult{edges, fields}` (WU-3C2; edges + fields in two idempotent passes) — also `write_resolves_to(neo, attempt_id, specs) -> int` (idempotent `MERGE` of `RESOLVES_TO`; empty → 0 without a session; raises `ResolutionUnavailableError(stage='write_resolves_to')`) and `persist_resolution_fields(neo, attempt_id, specs) -> int` (idempotent `SET`; `stage='persist_fields'` on failure); pure mapping seams `resolved_node_to_edge_spec`/`resolved_node_to_field_spec`.
- `apollo.graph_compare.validate_student_graph(student_graph: KGGraph) -> None` / `validate_reference(problem: dict) -> None` (WU-4A1; the §6.4 step-4 + §6.6 raw-graph gates — return `None` on a valid graph, else raise `StudentGraphInvalidError`/`ReferenceGraphInvalidError` carrying a `reasons` tuple; the reference side delegates to `validate_reference_graph`).
- `apollo.graph_compare.build_student_canonical(student_graph: KGGraph, resolution: ResolutionResult) -> CanonicalGraph` (WU-4A1; S_norm — merge student nodes by resolved key, normalize edges to canonical endpoints, DROP unresolved-endpoint edges (counted in `dropped_edge_count`), RETAIN unresolved nodes as `unresolved_nodes` findings-input; pure + deterministic, takes an ALREADY-computed `ResolutionResult` so the core stays LLM-free).
- `apollo.graph_compare.build_reference_canonical(problem: dict) -> ReferenceGraph` (WU-4A1; R_norm — per-problem `reference_solution` nodes + `depends_on` dependency DAG + one `ReferencePathView` per `declared_paths` entry; validates the reference FIRST (raises `ReferenceGraphInvalidError`); reads the problem's OWN reference_solution, NEVER the deduped entity payload).
- `apollo.graph_compare.build_problem_candidates(problem: dict, misconceptions: dict, *, canon_key_by_canonical_key: dict[str, int]) -> ProblemInputs` (WU-4A1; assembles the closed candidate set (REUSING `build_candidate_set`/`candidates_from_reference_solution`/`candidates_from_misconceptions`) + reads the per-problem `symbolic_mappings` as a NEW dict (default `{}`). This is the seam that plumbs the per-problem table into `resolve_attempt(student_graph, inputs.candidates, symbolic_mappings=inputs.symbolic_mappings)` — WU-4A1 does NOT call the resolver itself; the caller/WU-4C does).
- `compute_coverage(student_graph: KGGraph, reference_graph: KGGraph) -> dict` (async; LLM).
- `compute_rubric(coverage, reference_nodes, *, misconception_scores=None) -> dict` (pure).
- `generate_diagnostic(*, coverage, reference_steps, problem_text, rubric, model=None) -> str` (LLM).
- `draft_reply(history, kg_summary, *, problem_text=None, model=None, history_summary=None) -> str`.
- `Problem.to_kg_graph(attempt_id) -> KGGraph`; `load_problem(path)`; `load_concept(subject_id, concept_id)`.
- `compute_xp_earned(*, overall_score, difficulty, is_reattempt) -> int`; `apply_xp(*, db, user_id, xp_delta)`.

Core types (`apollo/ontology/`):
- **Node types** (Pydantic discriminated union on `node_type`): `equation` (`symbolic`, `label`, `variables`), `condition` (`applies_when`), `simplification` (`applies_when`, `transformation`), `definition` (`concept`, `meaning`), `variable_mapping` (`term`, `symbol`), `procedure_step` (`action`, `purpose`). Common fields: `node_id`, `attempt_id`, `source` (`parser|reference|system`), `parser_confidence` (default 1.0), `status` (`ACCEPTED|DISPUTED|DUAL`), `student_belief` (str|None).
- **Edge types** (`EdgeType` StrEnum) with allowed-pair validation: `PRECEDES` (proc→proc; ordering — there is no `order` int on nodes), `USES` (proc→equation), `DEPENDS_ON` (any→any, no self-loops), `SCOPES` (simplification/condition→equation). Common field `provenance` (`explicit|inferred`, default `explicit`; WU-2A): the parser tags LLM-emitted edges, deterministic-fallback and reference-graph edges keep the `explicit` default. As of WU-2A the parser emits all four edge types (SCOPES and DEPENDS_ON were previously dead / reference-graph-only); cross-turn edges are produced when a `graph_context` is supplied.
- **Coverage result**: `{per_step: {ref_id: "covered"|"missing"}, procedure_scores: {ref_id: 0..1}, confidences: {ref_id: 0..1}, negotiation_counts: {dual, disputed, paraphrased, skipped}}`.
- **Rubric result**: `{overall: {score, letter}, procedure|justification|simplification|misconception_corrected: {score, letter, present[, detected, resolved]}}`. Letter bands A+..F (`rubric.LETTER_BANDS`).
- **Done response**: `{rubric, diagnostic_narrative, coverage, progress: {xp_earned, xp_before/after, level_before/after, level_up, title_after, level_progress_pct, xp_to_next_level}, ...flat legacy XP fields}`.
- **Canonical-space types** (`apollo/graph_compare/canonical.py`, WU-4A1; all `@dataclass(frozen=True)`, all collection fields tuples): `CanonicalNode{canonical_key, node_type, source_node_ids, evidence_spans, symbolic=None, method=None, confidence=None}` (one canonical node — for S_norm the merge of all student nodes resolving to `canonical_key` carrying both raw node-ids + per-source evidence spans; for R_norm one reference step with empty evidence and `None` method/confidence); `CanonicalEdge{edge_type, from_key, to_key, provenance}` (a normalized edge whose endpoints are canonical keys); `CanonicalGraph{nodes, edges, unresolved_nodes, dropped_edge_count}` (S_norm — `unresolved_nodes` is a tuple of `(raw node_id, surface_text)` findings-input, `dropped_edge_count` is the count of edges dropped for an unresolved endpoint); `ReferencePathView{canonical_keys}` (one declared acceptable solution path as an ordered key tuple); `ReferenceGraph{nodes, edges, paths}` (R_norm — reference nodes + the `depends_on` DAG + one path view per `declared_paths` entry, `paths` length ≥ 1); `ProblemInputs{candidates, symbolic_mappings}` (`apollo/graph_compare/problem_inputs.py` — the closed candidate set + the per-problem symbolic-mapping table fed to `resolve_attempt`).

## Main data flows

**(a) Student teaching turn — `POST /sessions/{id}/chat` (`handlers/chat.py`)**
1. Load session + latest `ProblemAttempt` for `session.current_problem_id`; resolve the concept via `curriculum_db.load_concept_definition(db, concept_id=sess.concept_id)` (DB, course-scoped through the `concept_id` FK).
2. Intent state machine (checklist item #5): if `sess.pending_intent == "done"`, treat the message as a confirmation (`detect_confirmation` regex). Affirmed → dispatch `handle_done` inline and return a chat-shaped envelope with `intent_executed`; rejected/ambiguous → clear pending, fall through. Otherwise classify intent (`classify_intent`, cheap LLM); a non-teaching intent at confidence ≥ 0.7 sets `pending_intent` and returns a confirmation prompt without parsing.
3. Build cross-turn context then parse: chat reads the CURRENT subgraph (`store.read_graph` — the new turn's nodes aren't written yet, so it IS the prior-turns graph) and `build_graph_context` projects it into a `GraphContext` threaded into `parse_utterance` (ONE strict-json_schema GPT-4o call, temp 0.0) → typed nodes (`node_id = "stu_" + uuid12`) + all four typed edges (provenance-tagged), now cross-turn live (WU-2B). The deterministic USES (from `uses_equation_ordinals`, resolved through the ORIGINAL-entry-index map) + PRECEDES (consecutive procedure steps) derivation remains the no-context fallback used when the model emits no edges.
4. Write nodes then edges via `KGStore` (edges MUST be written after nodes — endpoints must exist at CREATE time). `write_nodes` de-dups cross-turn re-assertions by id (a prior id is reused, not cloned). `write_edges` no longer relies on Neo4j silent-drop: it validates endpoint existence + EDGE_ALLOWED_PAIRS BEFORE the CREATE and logs any drop/invalid (`write_edge_rejected` + a `write_edges` summary). Writes are rejected with `SessionFrozenError` if phase ∈ {PROBLEM_REVEAL, SOLVING, REPORT}.
5. Reply: `load_windowed_history` (last 12 turns raw + rolling cheap-LLM summary refreshed every 8 older turns, persisted on `apollo_sessions.history_summary`) → `summarize_for_apollo` (bullet list of the KG) → `draft_reply` with ONLY the problem text + KG summary + history. v1: no sufficiency / misconception / OLM-invite signals and no output filter on this path.
6. Persist the (student, apollo) message pair in one commit; return reply + full KG dump.

**(b) "I'm done teaching" — `POST /sessions/{id}/done` (`handlers/done.py`)**
1. Read the per-attempt graph BEFORE freezing. If `APOLLO_DONE_GATE_ENABLED` (default off): flag parser-sourced entries with `status == DISPUTED` or `parser_confidence < 0.6` that have no row in `apollo_kg_negotiations` → raise `ReviewRequiredError` (422, FE renders a review modal).
2. `store.freeze(session_id)` (phase → PROBLEM_REVEAL), then phase → SOLVING.
3. Reference graph: `problem.to_kg_graph(attempt_id)` from the `reference_solution` carried in `apollo_concept_problems.payload` (DB; resolved via `_find_problem(db, sess.concept_id, sess.current_problem_id)`, NOT `problems/problem_*.json`) — this is the v1 truth source (no retrieval/RAG).
4. `compute_coverage`: all matcher calls run concurrently (`asyncio.to_thread` + `gather`, no `return_exceptions`). Binary types (equation/condition/simplification) are batched one LLM call per type; procedure steps get one partial-credit call each (score ≥ 0.5 → "covered"). DUAL nodes with `student_belief` are graded against the student's wording. Transient failures retry 3x with backoff then raise `CoverageGradingError` (503 — grading unavailable, never a silent downgrade).
5. `compute_rubric` (pure): procedure 0.57 / justification 0.2375 / simplification 0.1425 / misconception_corrected 0.05; absent axes redistribute weight proportionally (no misconceptions → exact pre-P2.8 60/25/15). Misconception scores are read from `apollo_messages.metadata` — since the v1 chat path no longer writes per-turn metadata, this axis is normally absent.
6. `generate_diagnostic` (LLM) narrates the rubric verdict — hard-ruled not to contradict per-axis scores, formative tone.
7. Re-attempt detection (`attempt.result` already set, or `has_prior_graded_attempt` cross-session) → `compute_xp_earned` (score × difficulty multiplier 1.0/1.5/2.0 × 0.25 if re-attempt) → `apply_xp` updates `apollo_student_progress` and computes level (5 tiers, thresholds 0/300/800/1600/3000). Attempt gets `result="graded"`, `solver_trace=None`, `diagnostic_report={narrative, rubric, coverage}`; phase → REPORT. The `result` allowlist (`models.ATTEMPT_RESULTS`, enforced by the DB CHECK from migration 009+025: `solved/stuck/skipped/returned_to_hoot/abandoned/graded`) must include every value handlers write — `done.py` writes `graded`, `next.py` writes `abandoned` on a mid-problem switch. `has_prior_graded_attempt` counts `GRADED_ATTEMPT_RESULTS` (all of them except `abandoned`, which is a switch, not a grade).

**(c) Hoot handoff — `POST /sessions/from_hoot` (`hoot_bridge/session_init.py`)**
Bearer token → `require_user` resolves `user_id`; `require_course_member` checks membership for the given `search_space_id`. `curriculum_db.list_course_concepts(db, search_space_id)` builds the course's candidate concepts → `infer_concept_id` (GPT-4o, must return one of those candidate `concept_id`s or 409 `no_matching_concept`; a course with no curriculum → empty candidates → null → 409) → `select_problem(db, concept_id, difficulty)` (DB; deterministic: sorted by `Problem.id`, unattempted at difficulty, else `PoolExhaustedError`) → end stale active sessions → create `ApolloSession` (phase TEACHING, `user_id`, `search_space_id`, `concept_id` populated — `concept_cluster_id` no longer written) + `ProblemAttempt` → return `{session_id, attempt_id, problem}`.

## Non-obvious conventions

- **NO FALLBACK policy**: every failure mode is a named exception in `apollo/errors.py` with its own HTTP status + `error_code` registered in `api.py` (`register_exception_handlers`). Never soft-fail a grade or silently truncate context. Exceptions: deliberately soft-failing helpers — triviality classifier (→ trivial), intent classifier (→ "teaching"), history summarizer (→ no summary), LaTeX render (→ None). **WU-3C1 adds two named errors that are deliberately NOT registered as HTTP handlers in this unit** (api.py is out of WU-3C1 scope; both surface on non-request / post-grade paths): `CanonProjectionError(stage, last_error)` — raised by the `:Canon` seeder (`load_entities`/`merge_canon`), surfaced via the operational CLI's non-zero exit, never leaving a partial authoritative `:Canon` graph; and `RetentionError(attempt_id, last_error)` — raised by `stamp_graded_at`, which runs AFTER the grade/XP commit, so a failure surfaces (loud, never swallowed; a raw 500 if it ever reaches a request) WITHOUT voiding the durable grade — the next Done/retry/janitor re-stamps idempotently. **WU-3C2 adds two more named errors, also deliberately NOT registered as HTTP handlers here** (api.py is out of WU-3C2 scope; WU-4A wires the Done route that triggers resolution and registers both): `ResolutionUnavailableError(stage, last_error)` — resolver INFRASTRUCTURE failure (`stage ∈ {llm_adjudication, write_resolves_to, persist_fields}`), raised when the one LLM adjudication call or a Neo4j `RESOLVES_TO`/field write fails; like `RetentionError` it runs AFTER the grade commits, so it surfaces loud but must NOT void the grade — the caller sets `learner_update_pending=true` and the next Done/janitor retry re-runs the idempotent resolution; and `ResolutionInvalidOutputError(returned_key, allowed_keys)` — the HARD case where the one LLM adjudication call returns a key NOT in the closed candidate set (a hallucination); the resolver never fabricates a target. A per-node non-match is NOT an error — it is `unresolved` DATA (no edge, no event, logged; the unresolved rate is a parser-recall / curriculum-gap metric). **WU-3D adds one more internal named error, also NOT HTTP-registered** (matching the WU-3C1/3C2 precedent): `ConceptNotFoundError` (in `subjects/curriculum_db.py`) — raised by `load_concept_definition` when a session's `concept_id` points at a deleted `apollo_concepts` row (a should-never-happen given the `ON DELETE RESTRICT` FK), so it surfaces loudly (raw 500 if it ever reaches a request) rather than silently picking a default concept. **WU-4A1 adds two more, also deliberately NOT HTTP-registered here** (the Done route that triggers grading is WU-4C, which registers them): `StudentGraphInvalidError(reasons: tuple[str, ...])` — the §6.4 step-4 gate (`validate_student_graph`) fails on a raw-student grammar violation (illegal edge endpoints re-derived from node types, a missing endpoint, mixed `attempt_id`, or a PRECEDES cycle); and `ReferenceGraphInvalidError(reasons: tuple[str, ...])` — the §6.6 gate (`validate_reference` / `build_reference_canonical`) fails because the reference graph fails the §6.1 contract (`validate_reference_graph(...).errors` carried verbatim), which blocks grading. Both carry a structured `reasons` tuple and are raised loudly, never swallowed. The two user-facing 409s on the from_hoot path are unchanged: `NoMatchingConceptError` (now raised by `infer_concept_id` on null/unknown/invalid-JSON, including the empty-candidates "course has no curriculum" path) and `PoolExhaustedError` (its `concept_cluster_id` constructor/JSON field is RETAINED for API back-compat — now passed `str(concept_id)`).
- **Temperature**: 0.0 for every classification/extraction/grading call; 0.7 only for `draft_reply` (Apollo's persona).
- **Ignorance prompt invariants** (`APOLLO_SYSTEM_PROMPT` in `agent/apollo_llm.py`): Apollo never names concepts/laws the student hasn't named, never corrects the student, paraphrases using the student's exact vocabulary, stays in role under prompt injection, answers "do you know X?" with "no", keeps replies 1-3 sentences, and is a *stuck student* (asks about the plan, not the subject). v1 enforces anti-leak **structurally** — the reply call sees only problem text + student KG + history — instead of via the (now unwired) `output_filter`/`leakage_judge` two-stage filter; the filter code and `forbidden_named_laws.json` per-concept stoplists (fuzzy match ratio 0.8, min len 5, possessive stripping) remain for reference.
- **Canonicalization stance (v1)**: the parser captures the student's *own form* — the "Extract → Canonicalize" redesign (`2026-06-09-apollo-kg-extract-canonicalize-handoff.md`) was **superseded** by diff-at-Done, which removed the consumers that needed canonical strings (filter + solver). Coverage grades "by meaning, not wording".
- **Neo4j shape**: every node gets its type label (`Equation`, `Condition`, ...) PLUS secondary `:_KGNode` label so one index covers all subgraph reads/cleanup. Cypher can't parameterize labels → one CREATE template per label/edge type. `None` properties are omitted (Neo4j has no NULL). `_record_to_node` defensively coerces unknown `status` to ACCEPTED and defaults missing `parser_confidence` to 1.0 (protects pre-P1/P3 legacy nodes from false-firing gates). **WU-3C1 adds two things to the shape.** (1) Layer-2 nodes carry server-injected scoping/timestamp metadata — `user_id`/`search_space_id` (optional, `None`-omitted) + always `created_at`, and `graded_at` once stamped at Done; `_record_to_node` STRIPS all four before content reconstruction (they are metadata, not content), so the FE-visible KG payload is byte-identical. (2) A NEW `:Canon` label (NOT a `:_KGNode`) holds the course's Layer-1 skill inventory projected from Postgres, keyed on `key = apollo_kg_entities.id` (the surrogate id, deliberately NOT a `canonical_key`-derived key — two concepts that both mint the same `canonical_key` must stay TWO distinct `:Canon` nodes or every future `RESOLVES_TO` edge cross-contaminates). `:Canon` is rebuildable from Postgres (`canon_projection.project_canon`), never authored in Neo4j. **WU-3C2 adds the resolution shape.** Each `:_KGNode` gains four server-side resolution node-fields once the §5 resolver has run — `resolution` (`resolved`/`unresolved`/`ambiguous`), `resolved_key`, `resolution_method`, `resolution_confidence` — written by `resolution_store.persist_resolution_fields`; `_record_to_node` STRIPS all four on read (metadata, not content) so the FE-visible KG payload stays byte-identical (`resolved_key` is `None`-omitted when unresolved). And a NEW `(:_KGNode)-[:RESOLVES_TO {method, confidence, resolved_at}]->(:Canon)` CROSS-LABEL edge links each resolved student node to its canonical target (idempotent `MERGE` on the endpoint pair). `RESOLVES_TO` is written by `resolution_store`'s own Cypher and is intentionally NOT in `EdgeType`/`EDGE_ALLOWED_PAIRS` (those govern within-subgraph `:_KGNode→:_KGNode` edges).
- **Confidence thresholds form a ladder**: cheap-LLM judges act at ≥ 0.6 (triviality, leakage); intent confirmation gate at ≥ 0.7; OLM clarification invite at parse-confidence < 0.7; Done-gate (final brake) at < 0.6; coverage binary matches with confidence < 0.5 are downgraded to "missing" and logged `coverage_uncertain`.
- **Negotiable OLM (P3)**: moves mutate the Neo4j node (`status`, `student_belief`) AND append an immutable audit row to `apollo_kg_negotiations`; the Done-gate clears once each flagged entry has ≥ 1 move. DUAL + `student_belief` redirects coverage to grade the student's wording (for procedure steps it *replaces* `action`).
- **Freeze discipline**: writes/moves check phase via Postgres before touching Neo4j; reads are allowed while frozen.
- **Retention (§7, WU-3C1)**: `handle_end` now PERSISTS per-attempt subgraphs (it no longer deletes — it only marks the session `ended` in Postgres); `done.py` stamps `graded_at` on the frozen graph as the final, idempotent, post-commit retention write (Δt-anchoring in Layer-3 reads the stored value, never `now()`); `restart_problem` is the ONE explicit student wipe that still calls `delete_subgraph`; `delete_subgraph` remains in `KGStore` as the future janitor's pruning primitive; the `attempt_id < 0` test-cleanup convention is unchanged.
- **`summarize_for_apollo` format is contract**: the bullet shapes (`- equation (label): symbolic`, `- variable: term → symbol`, procedure steps ordered via the PRECEDES chain) are Apollo's vocabulary mirror — see `apollo/agent/LEAKAGE_POLICY.md`; don't change casually.
- **§8A curriculum cutover (WU-3D) — landed**: the curriculum is now fully DB-resolved and course-scoped. The hard-coded `_AVAILABLE_CLUSTERS` constant, the `_CLUSTER_TO_CONCEPT` map / `cluster_to_concept` / `list_problems_for_cluster` functions, and `infer_concept_cluster` are DELETED; the runtime resolves candidate concepts from `apollo_concepts WHERE …search_space_id`, picks problems from `apollo_concept_problems` by `concept_id`, and populates `apollo_sessions.concept_id`. Migration 027 dropped the legacy `apollo_sessions.concept_cluster_id` column (ORM column dropped in lockstep). `sympy_exec._CANONICAL_SYMBOLS` is still a fluid-mechanics list (out of WU-3D scope). `apollo_sessions.student_id` was dropped in migration 023 (replaced by `user_id UUID` referencing `auth.users`).
- **Tests**: nearly all `*/tests/` modules are `pytest.mark.skip`-ed with "Tracked in claude_v3_checklist.md item 1; will be re-enabled in test-rewrite phase". ## Product context

Apollo implements **learning-by-teaching** (protégé effect): explaining a concept to a naive
agent forces the student to surface their own understanding; the system grades the *teaching*,
not answers to questions. Pedagogical commitments visible in code: Apollo never corrects the
student mid-session (misconceptions are detected silently and graded at Done); the diagnostic
report is formative ("you didn't walk Apollo through X"), never punitive; the Negotiable OLM
(challenge/paraphrase/skip, after Bull & Pain's Mr Collins and STyLE-OLM) lets the student
contest what the parser claims they said before being graded on it; XP/levels (Apollo
Apprentice → Archon) reward completion with reduced credit for re-attempts. Prefer smaller,
higher-leverage v1 shipments over feature-complete ones (project memory:
`feedback_apollo_pedagogy_choices.md`).
