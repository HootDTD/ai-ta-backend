## Process
1. Read the relevant existing files before writing anything
2. Summarize what currently exists and how it works
3. Propose an implementation plan with the files you'll create or modify
4. Identify any risks or side effects on the retrieval pipeline or citation logic
5. Wait for my approval of the plan before writing any code
6. After implementation, write unit tests using Supabase mock fixtures
7. Run the test suite and confirm it passes before marking done
